Política de Privacidad

Última actualización: 21 de marzo de 2026

La extensión Stacklume está diseñada con la privacidad como prioridad. No recopilamos, almacenamos ni compartimos datos personales de ningún tipo.

1. Qué datos procesa la extensión

Cuando el usuario decide guardar un enlace, la extensión accede a:

  • URL de la página activa o del enlace seleccionado.
  • Título de la página (etiqueta <title> o og:title).
  • Descripción (metaetiqueta og:description o description).
  • Favicon e imagen OG (URLs de <link rel="icon"> y og:image).

La extensión no accede a cookies, localStorage, sessionStorage, contraseñas, formularios, historial de navegación ni ningún otro dato de las páginas visitadas.

2. Dónde se almacenan los datos

  • Configuración de la extensión (URL del servidor, token de API, preferencias): se almacena en chrome.storage.sync, cifrado por el navegador y sincronizado solo entre dispositivos del mismo perfil de Chrome del usuario.
  • Enlaces guardados: se envían exclusivamente a la instancia de Stacklume configurada por el usuario (por defecto http://127.0.0.1:7879, la app de escritorio local). Los datos se almacenan en la base de datos SQLite local de la aplicación Stacklume en el dispositivo del usuario.

3. Servidores de terceros

La extensión no envía datos a ningún servidor de terceros. Toda la comunicación se realiza entre el navegador del usuario y su propia instancia de Stacklume.

4. Telemetría y análisis

La extensión no contiene código de telemetría, analíticas, seguimiento (tracking), píxeles de rastreo ni herramientas de terceros como Google Analytics, Sentry, Mixpanel o similares.

5. Token de API

El token de API (necesario para guardar enlaces directamente) se almacena en chrome.storage.sync. Este almacenamiento está cifrado por el navegador y solo es accesible por la propia extensión. El token nunca se registra en logs, no se muestra en consola y se transmite únicamente al servidor Stacklume del usuario mediante cabeceras HTTP Authorization.

6. Permisos del navegador

  • activeTab: leer URL y título de la pestaña activa al hacer clic en la extensión.
  • storage: persistir la configuración de la extensión.
  • contextMenus: ofrecer opciones de guardado en el menú contextual.
  • tabs: consultar pestañas abiertas para la función "guardar todas las pestañas".
  • scripting: extraer metadatos de la página (og:tags, favicon). No accede a cookies ni datos sensibles.

7. Actualizaciones

Esta política puede actualizarse para reflejar cambios en la funcionalidad de la extensión. Cualquier cambio significativo será comunicado en las notas de la versión.

Contacto: Si tienes preguntas sobre esta política, puedes abrir un issue en GitHub.
